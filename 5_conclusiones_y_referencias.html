5. Conclusiones y Referencias

Conclusiones

Una vez obtenida la base de datos que representa el esquema lógico relacional de la empresa, éste debe validarse frente a las transacciones de los usuarios. El objetivo de este paso es validar el esquema lógico para garantizar que puede soportar las transacciones requeridas por los correspondientes usuarios. Estas transacciones se encontrarán en las especificaciones de requisitos de usuario. Si todas las transacciones se pueden realizar, el esquema queda validado. Pero si alguna transacción no se puede realizar, seguramente será porque alguna entidad o tabla, relación o atributo no se ha incluido en el esquema.
